# KopTumbuh — SQL Schema Comparison

**Files compared:**
- `schema_ddl.sql` — raw SIMKOPDES dump (27 tables)
- `koptumbuh_updated_minimal_data_model.sql` — KopTumbuh MVP (27 core + 14 extension tables)

## Quick Comparison

| Aspect | `schema_ddl.sql` | `koptumbuh_updated_minimal_data_model.sql` |
|---|---|---|
| Role | Raw DDL dump from SIMKOPDES metadata | KopTumbuh MVP schema with extensions |
| Tables | 27 | 27 core + 14 new |
| Schema | `public` (none) | `koptumbuh` |

## Key Design Upgrades in koptumbuh

### Data Types
- `TIMESTAMP` → `TIMESTAMPTZ` (timezone-aware, stored as UTC)
- `REAL` → `NUMERIC(18,2)` / `NUMERIC(18,3)` (exact decimals)
- `INTEGER` → `BIGINT GENERATED BY DEFAULT AS IDENTITY` for surrogate keys

### Constraints Added
- Non-negative checks on all monetary & quantity columns
- Confidence scores: `BETWEEN 0 AND 1`
- Progress: `BETWEEN 0 AND 100`
- Date ordering: `tanggal_kadaluarsa >= tanggal_berlaku`
- Enumeration CHECKs on all status/type columns
- Conditional required-reference checks on `pengguna_koptumbuh`

### rat_koperasi Change
- `schema_ddl.sql`: financial reports stored as `TEXT` (file paths)
- `koptumbuh`: financial reports stored as `JSONB` (structured data)

### Automatic Timestamps
9 `BEFORE UPDATE` triggers auto-set `updated_at` or `diperbarui_pada` on:
`pengguna_koptumbuh`, `pemasok_koptumbuh`, `pelanggan_koptumbuh`, `artikel_pengetahuan`,
`mapping_integrasi`, `produk_koperasi`, `transaksi_penjualan`, `inventaris_produk`
